# 03 — Stakeholder Analysis

**VERA: Volunteer Emergency Response Alliance**

## 3.1 Stakeholder Overview

Stakeholders are individuals, groups, or organizations with an interest in VERA's success. This analysis identifies who they are, what they need, their influence on the project, and how the system serves them.

## 3.3 Primary Stakeholders

### Citizens / Victims
| Attribute | Detail |
|-----------|--------|
| **Role** | Report emergencies, request blood, seek shelter and aid |
| **Needs** | Fast help, simple interface, trustworthy responders |
| **Pain points** | Slow social media outreach, no status tracking |
| **VERA features** | Emergency requests, blood requests, incident reporting, notifications |

### Volunteers
| Attribute | Detail |
|-----------|--------|
| **Role** | Accept requests, join rescue missions, distribute relief |
| **Needs** | Verified identity, safe assignments, recognition |
| **Pain points** | No central opportunity listing, no certificates |
| **VERA features** | Verification, opportunity applications, certificates, assignments |

### Blood Donors
| Attribute | Detail |
|-----------|--------|
| **Role** | Register blood group, respond to urgent requests |
| **Needs** | Targeted alerts only when their blood type is needed |
| **Pain points** | Generic social media posts, no structured matching |
| **VERA features** | Donor registration, automated notifications, search by blood group |

### Hospitals & Blood Banks
| Attribute | Detail |
|-----------|--------|
| **Role** | Broadcast urgent blood needs, coordinate with donors |
| **Needs** | Location-specific alerts, fast donor coordination |
| **Pain points** | Manual phone chains, no visibility of nearby donors |
| **VERA features** | Blood requests, location search, NGO coordination |

### NGOs & Relief Organizations
| Attribute | Detail |
|-----------|--------|
| **Role** | Coordinate relief, manage resources, recruit volunteers |
| **Needs** | Central hub, resource tracking, underserved area visibility |
| **Pain points** | Uncoordinated field operations, duplicate relief efforts |
| **VERA features** | Resource tracking, coordination requests, campaigns, coverage monitoring |

### Donors (Financial / Material)
| Attribute | Detail |
|-----------|--------|
| **Role** | Contribute money, food, medicine, clothing, equipment |
| **Needs** | Transparency on how contributions are used |
| **Pain points** | No tracking after donation |
| **VERA features** | Donation management, fundraising campaigns, allocation records |

| Government agencies | National safety, policy alignment | Future integration |

| Emergency response teams | Field deployment | Coordination requests |
| Students | Volunteer experience, certificates | Opportunity applications |
| Educational institutions | Social initiative programs | Certificate verification |
| System administrators | Platform integrity | User verification, reports |
| Developers | System maintenance and evolution | Technical implementation |

## 3.5 Stakeholder Power / Interest Matrix

| | **Low Interest** | **High Interest** |
|---|------------------|-------------------|
| **High Power** | Government agencies | NGOs, Hospitals, Administrators |
| **Low Power** | General public observers | Citizens, Volunteers, Donors, Students |

**Engagement strategy:**
- **High power, high interest** — Close collaboration (NGOs, hospitals, admins)
- **High power, low interest** — Keep informed (government)
- **Low power, high interest** — Consult and involve (citizens, volunteers, donors)
- **Low power, low interest** — Monitor only

---

## 3.6 Stakeholder Requirements Summary

| Stakeholder | Top 3 Requirements |
|-------------|-------------------|
| Citizens | Submit emergencies quickly, track status, get verified help |
| Volunteers | Get verified, receive safe assignments, earn certificates |
| Blood donors | Register once, get relevant alerts only |
| Hospitals | Post urgent blood needs, find nearby donors |
| NGOs | Track resources, recruit volunteers, monitor coverage |
| Donors | Donate easily, see where funds go |
| Administrators | Verify users/requests, generate reports |
| Students | Find opportunities, receive certificates |

## 3.7 Rich Picture Summary

The VERA Rich Picture illustrates how citizens, volunteers, NGOs, hospitals, blood banks, donors, and emergency response organizations interact through the platform to provide emergency assistance and coordinate resources. All stakeholders connect through a single hub rather than isolated channels.
